## 프로젝트 소개

### OF-YOU: 당신만을 위한 맞춤형 쇼핑몰

OF-YOU는 개인의 선호와 스타일에 맞춘 상품을 추천하고, 쉽고 편리하게 쇼핑할 수 있는 맞춤형 쇼핑몰입니다.  
 사용자 경험을 극대화하고, 직관적인 인터페이스와 **데스크톱, 테블릿, 모바일**에서 최적화된 환경을 제공합니다.

- **애자일 방식으로 진행된 2주 프로젝트**

  - 유연한 계획과 점진적 개발을 통해 쇼핑몰 기능 구현
  - 주기적인 피드백과 개선 작업을 통해 완성도 향상

- **실제와 같은 쇼핑몰 제작을 목표**
  - 상품 목록 조회, 장바구니, 결제 등 실사용 사례를 반영한 기능 구현
  - 사용자 친화적인 UI/UX와 반응형 디자인으로 다양한 디바이스에서 최적화

## 협업전략 및 준비과정

### 우리 팀의 협업 플랫폼으로 Notion을 사용했습니다.

주요 기능은 팀플 활동 및 진행 상황을 관리하는 것이었으며, 각 팀원이 맡은 역할과 일정에 따라 내용을 업데이트하며 효율적인 협업을 기획했습니다.

 <img width="400" src="./public/images/notion.png"/>

### Git은 프로젝트의 중심적인 버전 관리 툴로 활용되었습니다.

- 이슈 관리: 초기 단계에서 구현해야 할 기능들을 하나씩 GitHub 이슈에 작성했습니다. 이를 바탕으로 작업을 나누어 해결하는 방식으로 진행했습니다.
- 풀 리퀘스트(PR): 팀원 간 협업 충돌을 최소화하기 위해 매일 1개의 PR을 병합하도록 규칙을 정했습니다. 덕분에 충돌이 발생하더라도 빠르게 해결할 수 있었고 작업 효율이 높아졌습니다.

 <img width="600" src="./public/images/git.png"/>

### 간트 차트를 통해 프로젝트의 일정과 각 팀원의 진행 상황을 시각적으로 공유했습니다.

예를 들어, "17~18일: 카테고리 기능 구현"처럼 일정과 담당 업무를 명확히 기재하여 팀원 모두가 현재 작업 상태를 파악할 수 있도록 했습니다. 이를 통해 업무 분배와 일정 관리를 체계적으로 수행했습니다.

  <img width="600" src="./public/images/gantchart.png"/>

### 저희는 코드 컨벤션과 커밋 메시지 컨벤션 규칙을 사전에 정의하고 이를 철저히 준수했습니다.

- 코드 컨벤션: 일관된 코드 스타일을 유지하여 가독성을 높였습니다.
- 커밋 메시지 컨벤션: 커밋 히스토리를 명확히 하기 위해 작업 유형별로 메시지를 규격화했습니다.
이 규칙 덕분에 팀원 간 코드 리뷰가 수월해졌고, 작업 이력을 쉽게 추적할 수 있었습니다.

  <img width="600" src="./public/images/conven.png"/>

## 디렉터리구조

### "기능 기반 디렉터리 구조"(Feature-Based Directory Structure)

컴포넌트, 상태 관리, 페이지, 라우팅, 유틸리티 등을 디렉터리별로 나누어 파일을 정리했습니다.

- 기능 중심 분류: 프로젝트를 구성하는 각 주요 기능(예: features, pages, components)에 따라 디렉터리를 나누고, 관련 파일들을 그룹화했습니다.
- 모듈화: 각 폴더가 독립적으로 동작하며 특정 기능이나 역할을 수행합니다.
- 확장성: 새로운 기능이 추가될 때 관련 폴더에 파일을 추가하거나 새로운 폴더를 만들어 쉽게 확장할 수 있습니다.
